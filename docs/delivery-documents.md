# Documentos que deben acompanar la entrega

## Minimo recomendado
1. `README.md`
2. `docs/technical-decisions.md`
3. `docs/checklist-requirements.md`
4. `docs/architecture.md`

## Que cubre cada uno

### README.md
Explica stack, instalacion, variables de entorno, seed, credenciales demo, endpoints y forma correcta de levantar el proyecto.

### technical-decisions.md
Defiende por que se eligio React 19, MUI, Prisma, SQLite, JWT, drag and drop e i18n.

### code-audit-report.md
Resume la limpieza tecnica final: build health, correcciones de TypeScript, i18n aplicado y criterio de entrega limpia.

### checklist-requirements.md
Facilita al evaluador contrastar el codigo con lo pedido en la prueba.

### architecture.md
Da contexto rapido de capas, flujo y responsabilidades.
